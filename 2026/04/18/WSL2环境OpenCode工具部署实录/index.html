WSL2环境OpenCode工具部署实录

1. WSL 2 环境搭建与镜像优化

参考教程：教程

1.1 内核更新与故障处理

若执行 wsl --update 失败，需访问官方 GitHub 仓库直接下载最新 Release 版本进行手动安装。

  • 官方下载地址： https://github.com/microsoft/WSL/releases

1.2 Ubuntu 镜像选择与下载优化

官方手动安装文档（Microsoft 官方链接）提供的分发包下载速度极慢且版本结构复杂。

  • 优化方案： 弃用官方链接，改用清华大学开源软件镜像站。
  • RootFS 系统包路径： https://mirrors.tuna.tsinghua.edu.cn/ubuntu-cloud-images/noble/current/

3. 故障排查：处理 SSL 连接超时

3.1 故障现象

执行安装脚本 curl -fsSL https://opencode.ai/install | bash 时报错：

3.2 原因分析

由于跨境网络限制，WSL 环境无法直接建立加密连接。需借用宿主机（Windows）运行中的 Clash Verge 代理。

3.3 解决方案

步骤 A：开启 Clash 的“局域网共享” 在 Clash Verge 的 Setting 页面中，确保 Allow LAN (允许局域网连接) 选项处于 On (开启) 状态。

步骤 B：配置 WSL 2 自动识别宿主机代理 修改 ~/.bashrc 文件，动态获取 Windows 虚拟网卡 IP（宿主机 IP）并建立代理开关：

注：修改完成后执行 source ~/.bashrc 使配置立即生效。

4. OpenCode 安装与版本验证

在 Clash Verge 代理生效（Proxy ON）状态下，执行安装流程：

  1. 安装命令： curl -fsSL https://opencode.ai/install | bash
  2. 环境变量刷新： 重启终端或运行 source ~/.bashrc。
  3. 版本验证： 执行 opencode --version。
    • 预期输出： 1.3.17 (或更高版本)。

5. 项目初始化与模型切换

针对当前工程目录进行初始化配置：

  1. 目录切换： cd [Project_Directory]
  2. 进入交互界面： 执行 opencode 进入 TUI（终端用户界面）。
  3. 模型切换： 在交互框内输入 /models。
  4. 选定模型： 切换至 MiniMax m2.5 free 模型（当前环境下更佳的免费推理方案）。
